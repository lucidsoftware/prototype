Software Engineering Process

Designers

Engineers

WHITEBOARD & SKETCH

Designers sketch, mockup, and design new concepts referencing our existing style guide inside particle.

WHITEBOARD & SKETCH

Engineers participate early in brainstorming sessions and provide valuable feedback, ideas, and insight.

PROTOTYPE

Designers build interactive prototypes using particle components. Designers code what is necessary to get valuable feedback. Often images are used within a prototype to fake it.

USER TESTING

New designs are given to users, and tested thoroughly. Through directed discovery designers identify new problems & opportunities.

REFINE PROTOTYPE

Designers finalize design decisions based on feedback from users, product managers, and engineers.

VALIDATE PROTOTYPE

Engineers clarify anything missing in the prototype, compare to existing product, and offer feedback.

REFERENCE PROTOTYPE

Once the design has stablized, engineers are given an interactive prototype of the latest design.

VERIFY PRODUCTION CODE

Designers are involved during production and part of the code review. Verifying that production is in line with the prototype and current design guidelines.

BUILD PRODUCTION

Reusing HTML and CSS where it makes sense, engineers build a scalable, production-ready version of the prototype.

UPDATE STYLE GUIDE

Newly built components are added to particle design to be reused in the future.
